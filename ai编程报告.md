# Ai编程报告

## 使用的ai工具
- cursor
    - 形成大体框架
    - React/Next.js 代码生成
    - TypeScript 类型补全
    - Canvas 绘制逻辑实现
- chatGPT:
    - 技术方案设计
    - 调具体细节，如调整色彩自动适配
- kimi & 元宝: 
    - 国内ai来搜索色卡资料

## 工作流复盘
1. gpt给出方案设计，技术栈建议和对比分析
2. 写一个简单的大纲和要求丢给cursor形成工程框架
3. 调试运行，寻找可以优化的地方，然后让gpt或者cursor进行优化
4. 在项目基本可以成功运行以后开始调整色卡、图纸生成质量
5. 最终软件测试，进行交互的全面筛查和优化
6. 根据自己的思路写Markdown文件，然后丢给ai润色
7. 注意git的本地/远程管理（因为这个项目较小，远程管理提交较少，本地很快就做完了）

## 难点分析
### 一、RGB 匹配导致巨大色差
#### 现象
项目最开始使用的是：RGB 欧氏距离，进行颜色匹配。虽然实现简单，但实际效果很差。

很多情况下：
- 肤色会偏灰
- 深蓝会被匹配成紫色
- 肉眼看接近的颜色，算法却判断差距很大

生成出来的拼豆图纸色差非常明显。

#### 最初 AI 的翻车点
一开始，我直接让 AI：优化颜色匹配算法  
AI 只是简单改成：**HSV** 或者 **加权 RGB**

#### 后续修正 Prompt
进一步补充上下文：
```
目前拼豆颜色与原图片肉眼感知差异巨大，
希望使用更符合人眼视觉的颜色距离算法，
并尽量接近专业图像处理软件的效果。
```
AI 随后开始建议：**CIELAB**和**CIEDE2000**等方案。

#### 最终项目采用：
**CIELAB + CIEDE2000**进行颜色匹配。  
并新增：  
**线性光（Linear sRGB）平均采样**，避免普通 RGB 平均产生偏暗问题。

### 二、色卡搜索和爬数据转json
我给的最初提示是：
```
帮我找到Perler、Artkal、Hama这三个拼豆牌子的色卡 比如rgb或者其他数值和色号的数据 最好可以有可以导入我的工程代码的json或者其他csv之类的方式方便我搞匹配 
```
在此基础上一直向国内ai提交新的要求，最后kimi给我了网站汇总和爬数据py源码，不过可以直接下载跑好的json文件。与原网站对比后发现无错误，即导入项目使用。
