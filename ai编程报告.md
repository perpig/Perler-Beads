# Ai编程报告

## 使用的ai工具
- cursor
    - 形成大体框架
    - React/Next.js 代码生成
    - TypeScript 类型补全
    - Canvas 绘制逻辑实现
- chatGPT:
    - 技术方案设计
    - 调具体细节，如调整色彩自动适配
- kimi & 元宝: 
    - 国内ai来搜索色卡资料

## 工作流复盘

1. 先使用 GPT 对项目进行整体方案设计，包括：
   * 技术栈选择
   * 前后端方案对比
   * 图像处理实现方式
   * PDF 导出方案等

2. 编写一个简单的大纲和功能需求，将其交给 Cursor，快速生成工程基础框架。

3. 本地调试运行项目，寻找存在的问题与可优化的地方，再不断通过 GPT 或 Cursor 进行局部优化与重构。

4. 在项目基本可运行后，重点开始调整：
   * 色卡匹配效果，下列难点一会详细说明
   * 拼豆图纸生成质量，如分辨度调整
   * 网格显示细节，如每五格变红加粗
   * 图片像素化效果，如控制豆不要太多、图不要太大

5. 最后进行整体软件测试，对：
   * 页面交互
   * 文件导出
   * 参数调整
   * 边界情况

   进行全面检查与优化。

6. Markdown 文档先按照自己的思路撰写，再交给 AI 进行润色与结构优化。

7. 开发过程中注意 Git 的本地 / 远程版本管理。由于本项目整体规模较小，因此主要以本地开发调试为主，远程仓库提交次数相对较少。

## 难点分析
### 一、RGB 匹配导致巨大色差
#### 现象
项目最开始使用的是：RGB 欧氏距离，进行颜色匹配。虽然实现简单，但实际效果很差。

很多情况下：
- 肤色会偏灰
- 深蓝会被匹配成紫色
- 肉眼看接近的颜色，算法却判断差距很大

生成出来的拼豆图纸色差非常明显。

#### 最初 AI 的翻车点
一开始，我直接让 AI：优化颜色匹配算法  
AI 只是简单改成：**HSV** 或者 **加权 RGB**

#### 后续修正 Prompt
进一步补充上下文：
```
目前拼豆颜色与原图片肉眼感知差异巨大，
希望使用更符合人眼视觉的颜色距离算法，
并尽量接近专业图像处理软件的效果。
```
AI 随后开始建议：**CIELAB**和**CIEDE2000**等方案。

#### 最终项目采用：
**CIELAB + CIEDE2000**进行颜色匹配。  
并新增：  
**线性光（Linear sRGB）平均采样**，避免普通 RGB 平均产生偏暗问题。

### 二、色卡搜索和爬数据转json
我给的最初提示是：
```
帮我找到Perler、Artkal、Hama这三个拼豆牌子的色卡 比如rgb或者其他数值和色号的数据 最好可以有可以导入我的工程代码的json或者其他csv之类的方式方便我搞匹配 
```

在此基础上，我不断向国内 AI 提交新的需求与补充说明。  
后续：
- Kimi 提供了相关网站汇总
- Kimi 帮助分析网页结构,开浏览器开发者模式(F12)扒源码看怎么获取色卡得到json更好
- 最终生成了用于爬取色卡数据的 Python 脚本

同时，也可以直接下载已经整理好的 JSON 文件。
